# PHP 기말고사 실기 평가

## 개요

이번 실기 평가는 수업에서 학습한 PHP, MySQL, 데이터베이스 기반 웹사이트 개발 내용을 바탕으로 진행됩니다.

실기 평가는 **오픈북(Open Book)** 형태로 진행됩니다.

학생들은 제공된 CMS 프로젝트를 실행하고, 누락된 `article.php` 파일을 완성한 후 정상적으로 동작하도록 만들어야 합니다.

---

# 제공 자료

GitHub 저장소에는 다음 자료가 포함되어 있습니다.

* 전체 CMS 프로젝트
* 데이터베이스 파일
* 이미지 파일
* `article_TODO.php`
* `article.php` 전체 코드가 포함된 PPT 이미지

단, 실제 `article.php` 파일은 제공되지 않습니다.

학생은 `article_TODO.php`를 참고하여 `article.php`를 완성해야 합니다.

```
첨부된 PDF 파일에서 필요한 모든 코드와 설명을 확인하세요.
파트 A, 파트 B, 파트 C만 입력하시면 됩니다.

파트 D는 이미 제공되어 있습니다.
```

## Part 2. 개념 질문 (20점)

다음 질문에 간단히 답하시오.

### 1. prepare() 와 execute() 를 사용하는 이유는 무엇인가?
SQL Injection을 방지하고, SQL문과 데이터를 분리해서 안전하게 DB 작업을 하기 위해 사용한다.
prepare()는 SQL문을 미리 준비하고, execute()는 실제 값을 넣어 실행한다.
---

### 2. INSERT 와 UPDATE 의 차이점을 설명하시오.
INSERT는 데이터베이스에 새로운 데이터를 추가할 때 사용하고,
UPDATE는 이미 존재하는 데이터를 수정할 때 사용한다.
---

### 3. 파일 업로드 시 MIME 타입, 확장자, 파일 크기를 검사해야 하는 이유는 무엇인가?
악성 파일 업로드를 막고, 허용된 형식의 파일만 저장하기 위해 검사한다.
또한 너무 큰 파일이 서버 용량이나 성능에 문제를 일으키지 않도록 파일 크기도 제한해야 한다.
---

### 4. session_start() 의 역할은 무엇인가?
PHP에서 세션을 사용하기 위해 호출하는 함수이다.
로그인 정보 같은 사용자별 데이터를 서버에 저장하고, 페이지가 이동해도 그 상태를 유지할 수 있게 해준다.
